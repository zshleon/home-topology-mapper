## Task
Closes or implements: `device-identity-merge`

## Summary
Improved the device identity logic to prioritize MAC addresses and handle IP conflicts more gracefully.

## Changes
- Refactored `find_existing_device` in `inventory.py` to use a MAC-first matching strategy.
- Added logic to prevent IP-only matching from hijacking devices that already have a different MAC assigned.
- Implemented an IP conflict resolution step in `upsert_scanned_devices`: if a device moves to an IP currently 'held' by another device in the database, the old owner is marked as offline.
- Added `backend/tests/test_inventory.py` with 3 test cases covering these scenarios.
